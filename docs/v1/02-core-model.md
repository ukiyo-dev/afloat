# 核心模型

Afloat 的核心是承诺与事实之间的关系。

整体链路：

```text
主题承诺
  ↓
时间承诺
  ↓
偏移覆盖
  ↓
事实层
  ↓
派生视图
```

## 主题承诺

主题承诺回答：

> 哪些事情不应该无声无息地从我的生活中消失？

线程是主题承诺。线程由 `Group / Item` 定位，可以由用户主动创建，也可以由日历事件自动聚类而来。

主题承诺可以带：

- `Group`
- `Item`
- `expectedMinutes`
- `deadline`

主题承诺不安排时间。它只声明某个主题应该被观察。

## 时间承诺

时间承诺回答：

> 我把某段时间承诺给了什么？

计划层承载时间承诺，包括三种时间：

- 理想
- 娱乐
- 休息

理想、娱乐、休息都是正当计划。Afloat 不把娱乐视为错误，也不把休息视为无效时间。

三种计划时间继承同一套计划事件结构。它们都可以使用标题协议、进入线程追踪、参与主题承诺，并进入 `expectedMinutes + deadline` 可行性分析。

继承同一结构不代表语义可以混用。

休息类型的承诺必须以恢复为目的。睡眠、午休、放空、恢复性散步可以是休息；如果散步的主要目的变成构思、产出或推进某个主题，它更接近理想，而不是休息。

## 偏移记录

偏移记录回答：

> 这段承诺如何被改写？

偏移层包括两种偏移：

- 外部偏移：原有时间承诺因外部现实而改变
- 内部偏移：原有时间承诺因自身选择而改变

偏移不是第四、第五种普通计划时间。偏移层用于覆盖计划层，生成事实层。

## 事实层

事实层回答：

> 时间最终实际流向了哪里？

事实层不是用户直接输入的数据，而是 Afloat 从计划层与偏移层计算得出的结果。

事实层保留五种结果：

- 理想兑现
- 娱乐兑现
- 休息兑现
- 外部偏移
- 内部偏移

偏移中的娱乐不并入主动娱乐。Afloat 关心的是承诺结构，因此主动娱乐和内部偏移必须分开。

## 派生视图

派生视图是结构化展示数据，不是事实源。

Afloat 每次更新后从事实源全量重算派生视图。线程视图、统计结果、错误列表和访客模式镜像都属于派生视图内容。

事实源只有：

- 外部日历事件缓存
- 日历语义映射
- 线程声明
- 每日笔记
- 公开设置

派生视图必须可以随时从事实源重新生成。

## 每日笔记

每日笔记是解释层，不是日历协议的一部分。

它回答：

> 用户如何解释当天的承诺、偏移与事实？

每日笔记存储在 Afloat 数据库中，而不是外部日历中。日历负责承诺与偏移，笔记负责反思与解释。

每日笔记不参与：

- 计划层
- 偏移层
- 事实层
- 承诺兑现率
- 线程统计
- 可行性分析

每日笔记可以设为私密或公开。v1 访客模式复用同一镜像界面，只隐藏 Threads 模块；笔记可见性后续可以继续细化。

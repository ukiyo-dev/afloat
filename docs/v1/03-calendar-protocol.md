# 日历协议

Afloat v1 使用多日历协议，不依赖事件级颜色。

用户可以准备一组语义日历。事件所属日历决定它在 Afloat 中的语义。

日历协议支持渐进绑定。用户不需要一开始就配置全部语义日历。

## 语义日历

计划层日历：

- 理想
- 娱乐
- 休息

偏移层日历：

- 外部偏移
- 内部偏移

颜色只是日历客户端里的视觉辅助，不是 Afloat 的事实依据。

## 渐进绑定

v1 允许用户只绑定部分语义日历。

最低启动条件：

- 至少绑定一个计划层日历：理想、娱乐或休息之一

可选增强：

- 绑定更多计划层日历后，计划分布更完整
- 绑定外部偏移后，可以观察外部现实如何改写承诺
- 绑定内部偏移后，可以观察自身选择如何改写承诺

缺失语义日历不会被视为错误。它只表示 Afloat 无法观察该类时间。

示例：

```text
只绑定休息日历：Afloat 可以先成为睡眠观察镜像
只绑定娱乐日历：Afloat 可以先观察主动娱乐结构
绑定理想+内部偏移：Afloat 可以观察理想承诺与内部偏移
绑定全部日历：Afloat 形成完整承诺观察系统
```

## 计划层

计划层写下时间承诺。

示例：

```text
理想：Afloat：线程承诺 1
娱乐：游戏开发：玩同类游戏 1
休息：睡眠
```

同一时间只能存在一个计划层事件。计划层内部重叠是协议错误。

三种计划类型共享结构，但语义不同：

- 理想：推进性投入
- 娱乐：主动奖赏或滋养性投入
- 休息：恢复性投入

休息事件必须以恢复为目的。若一个活动主要用于思考、构思、训练或产出，即使形式上是散步、躺着或听音频，也不应归为休息。

## 偏移层

偏移层记录承诺如何被改写。

示例：

```text
内部偏移：刷手机
外部偏移：临时会议
```

同一时间只能存在一个偏移层事件。偏移层内部重叠是协议错误。

## 跨层覆盖

计划层与偏移层允许重叠。偏移只通过时间重叠覆盖计划。

示例：

```text
20:00-22:00 理想：写作
20:30-21:00 内部偏移：刷手机
```

生成事实：

```text
20:00-20:30 理想兑现：写作
20:30-21:00 内部偏移：刷手机
21:00-22:00 理想兑现：写作
```

v1 不做邻接兜底，不根据前后一小时猜测关联。

## 无来源偏移

偏移事件可以没有重叠计划。

示例：

```text
15:00-16:00 内部偏移：刷手机
```

这类事件仍进入事实层与统计。Afloat 不特殊警告，也不追问它偏离了什么。它是协议上的冗余能力。

## 标题协议

标题用于线程追踪和质量标记。

支持格式：

```text
主标题：副标题 1
🌟主标题：副标题 2
❔主标题：副标题
```

规则：

- 中文冒号 `：` 前为 `Group`
- 中文冒号 `：` 后为 `Item`
- 没有中文冒号时，`Group = Item = 标题主体`
- 序号只认标题末尾阿拉伯数字
- `01` 等价于 `1`
- `🌟` 表示极佳
- `❔` 表示存疑
- 质量标记不参与标题匹配

v1 不支持英文冒号作为正式协议。

## 删除的语法

v1 不使用 `A【B】` 语法。

原计划由计划层事件表达，实际偏移由偏移层事件表达。标题只描述活动内容，不承担计划与实际的双重结构。

## 不属于日历协议的内容

每日笔记不通过全天日历事件记录。

v1 将每日笔记存储在 Afloat 数据库中，作为解释层。日历协议只承载计划层与偏移层。

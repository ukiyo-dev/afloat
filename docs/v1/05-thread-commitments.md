# 线程承诺与可行性

线程是主题承诺。

它回答：

> 哪些主题不应该无声无息地从我的生活中消失？

线程不是任务，不是 checklist，也不代表完成状态。

## 线程来源

线程有两种来源：

- 主动创建：用户在 Afloat 中声明一个主题承诺
- 自动发现：Afloat 从带序号的计划层事件中聚类发现

主动创建的线程即使没有事实推进，也显示为未启动。

自动发现从计划层事件产生。理想、娱乐、休息三种计划类型都可以进入线程追踪。

前端 Threads 模块展示的是活跃线程：当前未释放的 Group generation 中的 Item。活跃线程中可以包含已经被无序号活动关闭的 Item；只有整个 Group generation 释放后，它们才从活跃视图消失。

## 线程身份

线程由 `Group / Item` 定位。

`Item` 是承诺参数的最小单位。`expectedMinutes` 和 `deadline` 都下发到 `Item`，不直接挂在 `Group` 上。

`Group` 是聚合视图：

- Group 的 `expectedMinutes` = 其下所有 Item 的 `expectedMinutes` 之和
- Group 的 `deadline` = 其下所有 Item 的最晚 deadline
- Group 的已兑现、未来承诺、事实缺口、未计划缺口都由 Item 汇总得到

理想、娱乐、休息可以混合同一个 `Group / Item`。线程是主题容器，三者只是不同性质的时间投入。

示例：

```text
理想：游戏开发：战斗系统 1
娱乐：游戏开发：玩同类游戏 1
休息：游戏开发：恢复散步 1
理想：游戏开发：战斗系统 2
```

这些事件属于同一个主题承诺。

休息线程必须仍然是恢复性承诺。它可以与某个主题有关，例如围绕长期创作安排恢复，但它的主要目的必须是恢复，而不是构思或产出。

## 标题解析

```text
主标题：副标题 1
```

- `Group = 主标题`
- `Item = 副标题`
- `Sequence = 1`

没有中文冒号时：

```text
写作 1
```

解析为：

```text
Group = 写作
Item = 写作
Sequence = 1
```

质量标记 `🌟` 与 `❔` 不参与线程匹配。

## 序号规则

带序号事件可以开启或延续线程实例。

无序号事件可以收束已有线程实例。

规则：

- `写作 1` 开启线程实例
- `写作 2` 延续线程实例
- `写作` 收束已有线程实例
- 只有 `写作`，不自动生成线程
- 中间缺号不报错，只按时间推进
- 序号倒退是协议错误，但不阻断其他统计

无数字收束是派生判断，不是永久状态。用户修改历史标题后，派生视图全量重算，线程状态随之改变。

## 删除空线程

用户只能删除空线程。

空线程指：

- 由用户主动创建
- 尚未匹配任何过去事实兑现
- 尚未匹配任何未来计划
- 尚未产生任何自动发现实例

删除空线程表示用户撤回一个尚未进入时间的主题承诺。

一旦线程已经有事实历史或未来计划，Afloat 不允许删除它。历史承诺应当保留为事实的一部分。

有历史的线程只能通过无序号事件收束当前实例。

示例：

```text
写作：小说 1
写作：小说 2
写作：小说
```

最后一个无序号事件表示当前实例收束，但不删除历史。

如果未来再次出现：

```text
写作：小说 1
```

则开启新的线程实例。

## 线程统计

线程统计基于事实层，而不是原始计划层。

对于同一 `Group / Item`：

- 已兑现：事实层中的理想兑现 + 娱乐兑现 + 休息兑现
- 未来承诺：当前时刻之后的理想计划 + 娱乐计划 + 休息计划
- 外部偏移：覆盖该线程计划时间的外部偏移
- 内部偏移：覆盖该线程计划时间的内部偏移

被偏移覆盖的计划时间不计入已兑现。

## Deadline 与 expectedMinutes

Item 声明可以带：

- `expectedMinutes`：预期投入时长
- `deadline`：承诺时间边界

它们用于承诺可行性分析。

`expectedMinutes` 只由事实层中的已兑现时间抵扣。未来计划不抵扣，只用于预测未来是否已经承载该承诺。

用户可以在已有 Group 下独立添加 Item。添加 Item 不要求该 Item 已经在日历中出现；主动创建的 Item 即使没有事实推进，也应显示为未启动或未追踪。

Group 不直接编辑 expectedMinutes 或 deadline。若需要改变 Group 聚合结果，用户应编辑其下 Item。

## 可行性指标

核心指标：

```text
已兑现 = 事实层中该线程的理想+娱乐+休息兑现时间
事实缺口 = max(0, expectedMinutes - 已兑现)
未来承诺 = 当前时刻之后该线程的理想+娱乐+休息计划时间
未计划缺口 = max(0, expectedMinutes - 已兑现 - 未来承诺)
计划覆盖率 = 未来承诺 / 事实缺口
```

事实缺口为 0 只表示投入时间已经覆盖目标，不直接表示线程已兑现。

线程已兑现由同 Group + Item 的无序号活动决定；当该无序号活动已经成为历史时，该 Item 视为已兑现。如果已兑现但已兑现时间小于 expectedMinutes，前端显示低估比例，表示原计划完成时间被低估。

这些指标先在 Item 级计算，再汇总为 Group 视图。Group 视图用于看主题整体压力，Item 视图用于看具体承诺项。

如果存在 deadline：

```text
每日需安排 = 未计划缺口 / deadline 前剩余天数
```

## 可行性状态

可行性状态只描述承诺结构，不评价用户。

- 已兑现：该 Item 已被同名无序号历史活动关闭
- 已安排：事实缺口大于 0，且未计划缺口为 0
- 仍需安排：存在未计划缺口，但节奏仍可承载
- 节奏紧张：每日需安排接近近期日均承载
- 结构失衡：每日需安排超过近期日均承载
- 已过期：deadline 已过，且事实缺口大于 0

近期日均承载默认使用最近 30 天计划层兑现的日均时长，即理想兑现 + 娱乐兑现 + 休息兑现。

页面展示数字优先于评价：

```text
目标投入：60h
已兑现：20h
事实缺口：40h
未来承诺：15h
未计划缺口：25h
```

## 不做

线程 v1 不做：

- 完成按钮
- 子任务
- checklist
- 自动排程
- 到期提醒
- 逾期惩罚
- AI 建议
- 模糊匹配

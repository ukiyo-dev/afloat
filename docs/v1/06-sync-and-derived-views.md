# 同步与派生视图

Afloat 不拥有外部日历。外部日历是最高事实源，Afloat 本地保存一份可重算的镜像。

v1 只从 password/token-based CalDAV provider 同步数据。

## 本地保存的数据

v1 最小事实源：

- 日历源与语义映射
- 原始日历事件缓存
- 线程声明
- 每日笔记
- 公开设置

派生视图是结构化展示数据，不是事实源。

每日笔记存储在 Afloat 数据库中，不从外部日历同步。它属于解释层，不参与计划层、偏移层或事实层计算。

## 原始事件缓存

原始事件缓存保留外部日历事件的原始形态。

需要保存的信息包括：

- provider
- externalCalendarId
- externalEventId
- etag
- icalUid
- startAt
- endAt
- timezone
- title
- rawIcs
- deleted
- syncedAt
- providerUpdatedAt

原始事件缓存用于重新解析、规则升级后重算、debug 和派生视图生成。

## 同步模式

v1 支持两种同步：

- 近期同步
- 重新校准

### 近期同步

近期同步用于日常刷新。

默认范围：

```text
过去 30 天 + 未来 30 天
```

近期同步只更新范围内的本地事件缓存，不删除范围外缓存。

### 重新校准

重新校准用于全量同步。

触发场景：

- 首次绑定日历
- 用户手动重新校准
- 日历映射发生变化
- 解析规则版本升级
- 本地缓存异常

重新校准以外部日历为准，重建或校准全部本地事件缓存。

## 派生视图重算

每次更新后，Afloat 从本地事实源全量重算派生视图。

更新包括：

- 日历同步完成
- 线程声明变化
- 笔记变化
- 公开设置变化
- 解析规则版本变化

重算内容包括：

- 计划层
- 偏移层
- 事实层
- 协议错误
- 线程视图
- 承诺可行性
- 本人模式镜像数据
- 访客模式镜像数据

同步可以是范围同步，但派生视图语义上始终等价于从全部本地事实源全量计算。

## 派生视图类型

v1 使用派生视图，而不是业务对象缓存。

建议类型：

- 私密每日派生视图
- 私密范围派生视图
- 访客模式镜像派生视图

本人模式每日派生视图服务镜像页默认日视图窗口。本人模式范围派生视图服务自定义日期范围、最近 7d 和最近 30d 这类范围统计。

本人模式范围派生视图的日期范围按用户本地日历日计算。自定义开始日与结束日使用闭区间，两端日期都包含。

线程视图只是本人模式派生视图中的一个区块。Afloat 不存储独立线程派生表。

## 确定性

派生视图必须满足：

```text
相同事实源 + 相同规则版本 = 相同派生视图
```

派生视图可以丢弃并重建。

## 规则版本

解析和统计规则需要有版本号。

规则版本变化时，触发重新校准或至少触发从本地缓存全量重算派生视图。

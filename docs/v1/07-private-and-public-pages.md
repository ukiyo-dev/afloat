# 镜像页与可见性

Afloat 只有一个主展示系统：镜像页。

镜像页根据访问者身份进入不同可见性模式：

- 本人模式：已登录用户或本地授权访问，查看完整事实镜像
- 访客模式：未登录匿名访问，查看同一镜像界面，但隐藏 Threads 和 Rules 模块

公开与私密不是两套产品界面，而是同一镜像页在不同身份下的模块裁剪结果。v1 中访客模式隐藏 Threads 和 Rules 模块。

## 本人模式

本人模式需要登录或本地访问权限。

本人模式展示：

- 每日时间线
- 计划层事件
- 偏移层事件
- 事实层结果
- 承诺兑现率
- 维护率
- 协议错误
- 线程追踪
- 承诺可行性
- 用户规则
- 规则违约历史
- 每日笔记
- 原始标题

本人模式允许用户手动触发：

- 近期同步
- 重新校准
- 派生视图重算

如果用户只绑定部分语义日历，本人模式只展示已可计算的模块。缺失语义不显示为 0，也不作为错误提示。

## 主视图

镜像页的主视图是日视图滑动窗口。

默认窗口长度为 1 个用户本地日，默认展示今天。窗口步进为 1 个用户本地日。

主视图必须提供：

- 上一天快捷操作
- 下一天快捷操作
- 自定义开始日与结束日的范围统计控件
- 最近 7d 快捷范围
- 最近 14d 快捷范围
- 最近 30d 快捷范围

上一天将当前窗口整体向前移动 1 个本地日。下一天将当前窗口整体向后移动 1 个本地日。

自定义范围使用日历日闭区间，开始日和结束日都包含。若结束日早于开始日，则以结束日作为单日范围，即 `startDate = endDate`。最近 7d、14d 和 30d 分别表示今天和此前 6、13、29 天。所有日期边界都按用户本地时区计算。

本人模式主视图应优先呈现：

- 当前窗口或所选范围的事实分布
- 日投入及其范围 Daily Load 百分比
- 承诺兑现率
- 维护率
- 协议错误数量
- 最紧张或失衡的线程
- 正在冷却或沉寂的主题

Overview 的 Work、Entertainment 与 Rest 继续使用各自固定语义色。归属于 Threads 的活动使用原色实心填充，正式 Items 与 `---` 均属于 Threads；不归属于 Threads 的活动使用相同语义色与纸色混合的低密度填充。填充密度不改变现有活动标签，也不表达该活动是否正在推进 Daily Load。

主视图不使用鼓励或责备文案。

## 线程页面

线程页面是镜像页的本人模式细分视图，展示线程追踪。

每个线程至少展示：

- Group / Item
- 来源：主动创建、自动发现或两者皆有
- 已兑现
- 未来承诺
- 事实缺口
- 未计划缺口
- deadline
- 可行性状态
- 外部偏移
- 内部偏移
- 历史实例
- 空线程删除状态

线程操作区包含 `New Plan` 和搜索框。移动端下两者保持同一行：`New Plan` 不换行，搜索框占用剩余宽度。

当前搜索框过滤所选线程视图的 Group / Item，不改变顶部统计数字。线程视图提供 `Active / Inactive / All` 过滤：Active 只看当前活跃 Item，Inactive 只看已收束 Item，All 合并两者。

线程排序优先展示最需要被看见的承诺：

1. 已过期
2. 结构失衡
3. 节奏紧张
4. 仍需安排
5. 已安排
6. 已兑现
7. 无可行性字段但冷却或沉寂的线程

## 每日笔记

用户可以在同一个日期写多篇笔记。每篇笔记独立保存，并分别设置可见性。

每日笔记存储在 Afloat 数据库中，不使用全天日历事件。它是解释层，只用于记录用户对当天事实镜像的总结与反思。

笔记可以设为：

- 私密
- 公开

公开笔记会进入访客模式。私密笔记只在本人模式展示。

## 规则页面

规则页面是镜像页的本人模式细分视图，统一展示 `TEST` 与 `SIGNED` 用户规则。

规则页面展示：

- Active / Disactive / All 过滤
- `TEST` / `SIGNED` 承诺状态；不新增对应筛选或独立区域
- 规则名
- 规则内容
- 当前 run
- 历史 best run
- break 数量
- 违约历史
- 违约记录入口
- 停用入口
- `TEST` 规则的正式签署入口

当前过滤范围内 `TEST` 排在 `SIGNED` 之前。正式签署直接复用签署当下的 current run 起点与天数，并沿用 best 和违约历史，不在签署点重新起算。规则页面不展示补救、豁免、复活 run 或自动建议。

## 访客模式

访客模式通过匿名入口访问。

访客模式复用本人模式的主视图、范围控件、事实层、协议错误和笔记展示。

访客模式不展示：

- Threads 模块
- Rules 模块
- 线程追踪列表
- 线程编辑入口
- 线程可行性、deadline、expectedMinutes、承诺缺口
- 用户规则
- 规则违约历史
- 违约原因

访客模式仍可展示事实层时间线、原始标题、协议错误和每日笔记。它是低强度可见性，不是脱敏统计页。

如果用户只绑定部分语义日历，访客模式沿用本人模式的局部镜像规则。

访客模式可以展示协议错误详情，但不展示 Threads 派生出来的承诺压力或可行性。

## 访客模式原则

访客模式不是社交页，也不是成绩单。

它的作用是引入低强度、非互动、匿名的可见性，让用户知道自己的时间结构有一个公开镜像。

访客模式隐藏 Threads 和 Rules 模块，不隐藏事实镜像本身。

# 渐进式使用

Afloat 的完整形态是承诺观察系统，但用户不需要一开始就面对完整事实。

v1 支持渐进式使用：用户可以只启用一个语义日历，从自己当前能面对的部分开始记录。系统从已有数据生成局部镜像，并随着更多语义日历启用逐步展开。

完整协议是长期形态，不是启动门槛。

## 渐进原则

- 至少一个计划层日历即可启动
- 缺失语义不显示为 0
- 缺失语义不算错误
- 页面只展示当前可计算的模块
- 用户可以随时添加更多语义日历
- 新增语义日历后，派生视图从本地事实源全量重算

## 局部镜像

当用户只绑定部分语义日历时，Afloat 生成局部镜像。

局部镜像不是残缺状态，而是渐进使用中的合法产品形态。

缺失语义的含义是：

> Afloat 尚未观察这类时间。

它不表示：

> 用户没有这类时间。

## 只启用休息

当用户只启用休息日历时，Afloat 像一个恢复观察器。

它回答：

> 我的身体和精神恢复是否有稳定结构？

展示重点：

- 休息总量
- 休息分布
- 休息时段
- 休息线程
- expectedMinutes
- deadline
- 未来承诺
- 未计划缺口
- 维护率
- 每日笔记

不展示：

- 理想为 0
- 娱乐为 0
- 偏移统计
- 完整承诺兑现率

## 只启用娱乐

当用户只启用娱乐日历时，Afloat 像一个娱乐镜像。

它回答：

> 我的主动娱乐流向了哪些主题？

展示重点：

- 娱乐总量
- 娱乐时段
- 娱乐主题
- 娱乐线程
- 维护率
- 每日笔记

此时所有娱乐都是主动娱乐。没有偏移日历时，Afloat 不能判断娱乐是否来自偏移。

## 只启用理想

当用户只启用理想日历时，Afloat 像一个理想热度仪。

它回答：

> 哪些理想真的进入了时间？

展示重点：

- 理想投入
- 理想线程
- 发热、冷却、沉寂
- expectedMinutes
- deadline
- 未来承诺
- 未计划缺口

没有偏移日历时，Afloat 不能解释理想为什么没有兑现。

## 启用多个计划类型

当用户启用多个计划类型时，Afloat 像一个主动时间结构仪。

它回答：

> 我如何在不同时间性质之间分配承诺？

理想、娱乐、休息继承同一个计划事件结构：

- 同一套标题协议
- 同一套 `Group / Item`
- 同一套序号规则
- 同一套线程承诺
- 同一套 expectedMinutes 与 deadline 可行性分析

区别只在时间性质：

- 理想表示推进性投入
- 娱乐表示主动奖赏或滋养性投入
- 休息表示恢复性投入

同一个 `Group / Item` 可以同时包含理想、娱乐和休息。

示例：

```text
理想：游戏开发：战斗系统 1
娱乐：游戏开发：战斗系统 2
休息：游戏开发：恢复散步 3
```

线程页应同时展示合计与分项：

```text
总兑现：14h
理想兑现：8h
娱乐兑现：4h
休息兑现：2h
```

休息类型的承诺必须保持恢复语义。如果散步、躺卧、听音乐等活动的主要目的变成构思、训练或推进主题，它应记录为理想或娱乐，而不是休息。

## 启用完整计划层

当用户启用理想、娱乐、休息时，Afloat 像一个计划层镜像。

它回答：

> 我承诺的生活结构是什么？

此时可以展示完整计划分布，但仍不能解释承诺如何被改写，因为偏移层尚未启用。

## 启用计划层与偏移层

当用户启用至少一个计划层日历与至少一个偏移层日历时，Afloat 进入承诺观察系统形态。

它开始回答：

- 我承诺了什么？
- 我兑现了什么？
- 哪些承诺被外部现实改写？
- 哪些承诺被自身选择改写？
- 哪些主题正在变冷？
- 哪些承诺没有被未来时间承载？

完整五类日历启用后，Afloat 形成完整事实镜像。

## 实现要求

渐进式使用需要落实到页面与统计逻辑中：

- 未绑定语义不参与统计
- 未绑定语义不显示为 0
- 缺失模块直接隐藏
- 访客模式沿用局部镜像规则
- 新增语义日历后不需要迁移历史，只需重新同步并全量重算派生视图

渐进式不是 onboarding 技巧，而是 Afloat 的核心使用哲学：用户从自己能面对的地方开始，逐步扩大可见事实。
